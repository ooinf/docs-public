# Sites e Embeds

Sites permitem publicar listas em uma experiência organizada para um projeto, comunidade ou base de conhecimento.

## Site

Use um site quando quiser reunir várias páginas e listas em um endereço único.

Exemplos:

- portal de um projeto;
- catálogo público;
- base de consulta;
- repositório de materiais;
- página de uma comunidade.

## Planejar o site

Antes de publicar, defina:

- público principal;
- listas que farão parte do site;
- páginas de apoio necessárias;
- ordem do menu;
- visualização ideal para cada lista;
- quais conteúdos podem ser vistos por visitantes.

## Menu

Monte o menu com nomes curtos. Agrupe páginas e listas de forma previsível para que visitantes encontrem o conteúdo sem conhecer a estrutura interna.

Exemplo de menu:

```text
Sobre
Instituições
Projetos
Materiais
Contato
```

## Página e lista

Use páginas para textos estáveis, como apresentação, instruções e critérios. Use listas quando o conteúdo precisa de busca, filtros, relações e atualização frequente.

## Embed

Embed é uma versão enxuta de uma lista para incorporar em outro site.

Use embed quando os dados continuam no OOInfo, mas a consulta precisa aparecer em uma página externa.

## Escolher visualização para visitantes

| Conteúdo | Visualização indicada |
| --- | --- |
| Catálogo com muitas colunas | Tabela com filtros. |
| Diretório para descoberta | Cards. |
| Materiais com capa | Galeria. |
| Manual ou guia | Documento. |
| Categorias aninhadas | Hierarquia. |

## Antes de publicar

Confira:

- se as listas estão com a visibilidade correta;
- se o menu não aponta para conteúdo restrito;
- se a visualização escolhida funciona bem em celular;
- se imagens e links carregam corretamente.

## Revisão periódica

Depois de publicar, revise o site de tempos em tempos:

- links quebrados;
- listas vazias;
- descrições desatualizadas;
- imagens ausentes;
- permissões diferentes do esperado;
- conteúdo importante escondido em muitos cliques.
