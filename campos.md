# Campos

Campos são as colunas da lista. Cada campo define qual tipo de valor o item deve receber.

## Tipos comuns

| Tipo | Use para | Exemplo |
| --- | --- | --- |
| Texto | Valores curtos | Nome da instituição |
| Texto longo | Descrições maiores | Resumo do projeto |
| Texto rico | Conteúdo formatado | Manual, artigo ou guia |
| Número | Quantidades e métricas | Ano, nota, preço |
| Data | Prazos e eventos | Data de publicação |
| URL | Links externos | Site oficial |
| E-mail | Contato | contato@exemplo.com |
| Imagem | Logos, capas e fotos | Logo da organização |
| Arquivo | Anexos e documentos | Edital em PDF |
| Relação | Conexões com outra lista | Autor, categoria, cidade |
| Tags | Palavras-chave flexíveis | educação, cultura |
| Sim/Não | Estados binários | Publicado? |

## Como escolher o tipo

Escolha o tipo pensando no uso posterior, não apenas no preenchimento.

| Necessidade | Tipo recomendado |
| --- | --- |
| Filtrar por local, área ou status | Texto curto, seleção, tags ou relação |
| Ordenar por data ou valor | Data ou número |
| Guardar explicação longa | Texto longo |
| Criar uma página rica dentro do item | Texto rico |
| Conectar registros | Relação |
| Exibir capa, logo ou evidência visual | Imagem |
| Anexar comprovante ou material | Arquivo |

## Boas práticas

- Use nomes curtos e fáceis de entender.
- Evite duplicar informações em campos parecidos.
- Prefira campos específicos quando quiser filtrar ou ordenar.
- Use texto rico apenas quando a informação precisa de formatação.
- Use relações quando a mesma informação aparece em várias listas.

## Campos obrigatórios

Marque como obrigatório apenas o que é indispensável para entender o item. Campos obrigatórios demais dificultam contribuição.

Um bom ponto de partida:

- título ou nome do item;
- descrição curta quando o nome não basta;
- categoria ou tipo quando a lista é grande;
- link ou fonte quando a informação precisa ser verificável.

## Texto rico

O campo de texto rico permite títulos, listas, links, imagens e tabelas. Ele é ideal para descrições editoriais, tutoriais e documentos.

Para valores simples que serão buscados ou filtrados, prefira texto comum.

## Relações

Use relação quando o valor tem vida própria. Uma cidade, uma instituição, uma pessoa autora ou uma categoria complexa podem merecer listas separadas.

Relações ajudam a:

- evitar nomes duplicados com grafias diferentes;
- navegar entre registros conectados;
- filtrar por itens relacionados;
- reaproveitar uma mesma base em várias listas.

## Campos que costumam causar problema

| Sinal | Ajuste recomendado |
| --- | --- |
| Muitas variações para o mesmo valor | Transformar em seleção, tags padronizadas ou relação. |
| Texto longo usado para filtro | Separar em campos menores. |
| Campo quase sempre vazio | Tornar opcional ou remover. |
| Informação repetida em várias listas | Criar relação. |
| Campo com muitos formatos diferentes | Adicionar orientação de preenchimento. |
