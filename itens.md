# Itens

Um item é um registro dentro de uma lista. Em uma lista de livros, cada livro é um item. Em uma lista de instituições, cada instituição é um item.

## Criar item

1. Abra a lista.
2. Escolha inserir dados.
3. Preencha os campos obrigatórios.
4. Revise as informações.
5. Salve.

Dependendo das regras da lista, o item pode ser publicado imediatamente ou ficar aguardando revisão.

## Preenchimento consistente

Itens ficam mais úteis quando seguem o mesmo padrão. Defina pequenas convenções para a lista.

Exemplo:

| Campo | Padrão sugerido |
| --- | --- |
| Nome | Nome oficial, sem abreviação desnecessária. |
| Cidade | Nome da cidade sem complemento no mesmo campo. |
| Site | Link completo começando com `https://` quando possível. |
| Descrição | Uma ou duas frases objetivas. |
| Tags | Palavras curtas e reaproveitáveis. |

## Editar item

Edite um item quando precisar corrigir, complementar ou atualizar informações.

Se a lista usa revisão, a edição pode virar uma proposta antes de aparecer para todos.

## Evitar duplicidade

Antes de criar um item, pesquise pelo nome e por variações comuns. Duplicidades deixam filtros, relações e contagens menos confiáveis.

Se encontrar dois itens para a mesma coisa:

- mantenha o item mais completo;
- transfira informações úteis;
- preserve links e relações importantes;
- peça revisão quando não tiver certeza.

## Relações entre itens

Campos de relação conectam um item a outro. Isso evita repetir informações.

Exemplo: uma lista de livros pode se relacionar com uma lista de autores. Assim, cada livro aponta para o autor correto em vez de copiar o nome manualmente.

## Anexos e imagens

Use imagens quando elas ajudam a reconhecer o item. Use arquivos quando o material precisa ser baixado ou consultado separadamente.

Boas práticas:

- escolha imagens nítidas;
- use nomes de arquivo descritivos;
- evite arquivos muito grandes sem necessidade;
- confira se o material pode ser compartilhado publicamente.

## Qualidade dos dados

Antes de salvar, confira:

- nomes sem abreviações confusas;
- links funcionando;
- datas no formato correto;
- imagens adequadas;
- relações apontando para o item certo.

## Quando comentar em vez de editar

Use comentário quando houver dúvida, contexto ou discussão. Use edição quando a correção já estiver clara.

Comentários são úteis para:

- pedir fonte;
- explicar divergências;
- registrar uma observação temporária;
- chamar atenção para algo que precisa de revisão.
