# Colaboração e Revisão

O OOInfo permite que várias pessoas ajudem a manter listas atualizadas.

## Formas de colaborar

Usuários podem:

- sugerir novos itens;
- propor correções;
- comentar informações;
- avaliar itens quando a lista permite;
- participar de revisões quando convidados.

## Escolher o modelo de colaboração

Cada lista pode ter um nível diferente de abertura.

| Modelo | Quando usar |
| --- | --- |
| Somente equipe | Conteúdo fechado ou ainda em preparação. |
| Sugestões abertas | Comunidade pode contribuir, mas mudanças passam por avaliação. |
| Inserção direta | Boa para bases com baixo risco e colaboradores confiáveis. |
| Comentários | Útil quando o público deve apontar problemas sem alterar dados. |

## Revisão

A revisão protege listas colaborativas. Quando ativada, uma mudança enviada por um usuário pode ficar pendente até ser aceita.

Use revisão quando:

- a lista é pública;
- a qualidade dos dados precisa ser controlada;
- existe risco de informação incorreta;
- várias pessoas contribuem ao mesmo tempo.

## O que revisar

Ao avaliar uma contribuição, observe:

- se o item pertence ao tema da lista;
- se os campos obrigatórios estão completos;
- se não existe duplicidade;
- se links e fontes fazem sentido;
- se imagens e anexos são adequados;
- se a linguagem está clara e respeitosa.

## Votação

Algumas listas podem usar votação para decidir se uma contribuição deve entrar. Nesse caso, a mudança só é aplicada quando atinge o critério configurado pela lista.

## Papéis comuns

- **Dono da lista** define estrutura, regras e visibilidade.
- **Colaborador** adiciona ou sugere informações.
- **Revisor** confere contribuições pendentes.
- **Leitor** consulta e pode interagir conforme as permissões.

## Comentários

Comentários ajudam a discutir dúvidas e melhorar dados sem alterar imediatamente o item.

Use comentários para perguntas, contexto e justificativas. Use edição quando a informação final já estiver clara.

## Boas práticas de convivência

- Explique por que uma contribuição foi rejeitada.
- Prefira correções objetivas a discussões longas.
- Oriente novos colaboradores com exemplos.
- Use comentários para dúvidas e edição para mudanças confirmadas.
- Revise periodicamente itens muito acessados.
